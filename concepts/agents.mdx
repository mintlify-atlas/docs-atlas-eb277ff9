# Agents

Agents in ccprompts are specialized subagents that provide domain-specific expertise for complex tasks. Each agent follows a standardized 7-step methodology to ensure consistent, high-quality outputs across different domains.

## What Are Agents?

Agents are markdown files in `.claude/agents/` that define specialized behaviors, expertise areas, and systematic approaches for specific domains. They complement commands by providing focused, expert-level assistance.

<Note>
For more information about the subagent concept, see [Anthropic's documentation on sub-agents](https://docs.anthropic.com/en/docs/claude-code/sub-agents).
</Note>

## Agent Frontmatter

Every agent starts with YAML frontmatter defining its metadata:

```yaml
---
name: rust-expert
description: Use this agent when you need Rust programming expertise including ownership, lifetimes, unsafe code, or performance optimization. This agent specializes in memory safety, concurrent programming, and systems-level Rust development. Examples: <example>Context: The user is struggling with Rust ownership. user: "I'm getting lifetime errors when trying to return a reference from a function" assistant: "I'll use the rust-expert agent to help you understand lifetime annotations and fix your ownership issues" <commentary>Rust lifetime and ownership issues require deep understanding of the borrow checker.</commentary></example>
color: yellow
---
```

### Frontmatter Fields

<Accordion title="name">
The agent's identifier in kebab-case. Must match the filename without `.md` extension.

```yaml
name: rust-expert  # matches rust-expert.md
```
</Accordion>

<Accordion title="description">
A comprehensive description that enables auto-invocation. Must include:
- When to use the agent
- Specific capabilities
- Example usage scenarios with XML `<example>` tags
- Commentary explaining why the agent was selected

```yaml
description: Use this agent when you need [primary use case]. This includes [specific capabilities]. Examples: <example>Context: [scenario] user: "[request]" assistant: "[response]" <commentary>[explanation]</commentary></example>
```
</Accordion>

<Accordion title="color">
Visual categorization using plain English color names:

- **blue**: Development agents
- **green**: Operations agents  
- **yellow**: Data/AI agents
- **red**: Specialized agents
- **purple**: Creative agents
- **orange**: Business agents

```yaml
color: yellow  # Data/AI domain
```
</Accordion>

## The 7-Step Methodology

All ccprompts agents follow a standardized 7-step approach that ensures systematic, thorough problem-solving:

<Steps>
  <Step title="Analysis">
    Analyze the problem domain, context, and requirements. Identify specific considerations and constraints.
    
    **Example from rust-expert:**
    
    ```markdown
    1. **Master Ownership and Borrowing**: Navigate Rust's ownership system:
       - Ownership rules and move semantics
       - Borrowing and references
       - Lifetime annotations and elision
       - Smart pointers (Box, Rc, Arc, RefCell)
       - Interior mutability patterns
    ```
  </Step>

  <Step title="Identification">
    Identify key patterns, issues, or opportunities relevant to the task.
    
    **Example from performance-optimizer:**
    
    ```markdown
    2. **Apply Systems Thinking Approaches**: Understand interdependencies, 
       feedback loops, and emergent behaviors that affect overall system 
       performance and efficiency.
    ```
  </Step>

  <Step title="Action/Techniques">
    Apply domain-specific techniques and methodologies. This is typically the most detailed step with 3-4 technique categories.
    
    **Example from rust-expert:**
    
    ```markdown
    3. **Optimize Performance**:
       - Zero-cost abstractions
       - SIMD and vectorization
       - Memory layout optimization
       - Allocation strategies
       - Const evaluation and generics
       - Profile-guided optimization
    ```
  </Step>

  <Step title="Implementation">
    Provide implementation guidelines, requirements, and best practices.
    
    **Example from agent-orchestrate command:**
    
    ```markdown
    4. **Task Decomposition**: Break complex tasks into agent-specific subtasks
       - Complexity Analysis: Analyze task complexity and requirements
       - Skill Mapping: Map required skills to available agents
       - Dependency Detection: Identify task dependencies and ordering
       - Resource Allocation: Allocate computational resources optimally
    ```
  </Step>

  <Step title="Consideration">
    Address trade-offs, decision factors, and contextual considerations.
    
    **Example from performance-optimizer:**
    
    ```markdown
    5. **Balance Competing Objectives**: Optimize for multiple performance 
       dimensions including speed, quality, cost, scalability, and 
       maintainability while managing trade-offs effectively.
    ```
  </Step>

  <Step title="Validation">
    Define correctness checks, verification approaches, and quality assurance methods.
    
    **Example from rust-expert:**
    
    ```markdown
    6. **Build Robust Systems**:
       - Error handling with Result and Option
       - Custom error types
       - Panic handling and recovery
       - Resource management (RAII)
       - Testing strategies
       - Documentation with examples
    ```
  </Step>

  <Step title="Measurement">
    Establish metrics, tools, and validation methods to measure success.
    
    **Example from performance-optimizer:**
    
    ```markdown
    7. **Execute Continuous Improvement**: Design performance improvement 
       processes that enable ongoing optimization and adaptation to changing 
       requirements and constraints.
    ```
  </Step>
</Steps>

## Agent Template System

New agents are created using the `agent-template-wizard` and the `SUBAGENT_TEMPLATE.md` template file:

### Template Placeholders

The template includes placeholders that are filled during agent creation:

```markdown
---
name: {AGENT_NAME}
description: Use this agent when you need to {PRIMARY_USE_CASE}. This includes {SPECIFIC_CAPABILITIES}.
color: {AGENT_COLOR}
---

You are an elite {DOMAIN_EXPERT_TITLE} with deep expertise in {CORE_EXPERTISE_AREAS}.

When {PRIMARY_ACTION_VERB}, you will:

1. **{ANALYSIS_STEP_NAME}**: {ANALYSIS_DESCRIPTION_WITH_SPECIFIC_CONSIDERATIONS}

2. **{IDENTIFICATION_STEP_NAME}**: {WHAT_TO_IDENTIFY_AND_HOW}

3. **{ACTION_STEP_NAME}**:
   - {TECHNIQUE_CATEGORY_1}: {SPECIFIC_TECHNIQUES_AND_METHODS}
   - {TECHNIQUE_CATEGORY_2}: {SPECIFIC_TECHNIQUES_AND_METHODS}
   ...
```

### Creating a New Agent

<Steps>
  <Step title="Invoke the Wizard">
    Use the agent-template-wizard to start the creation process:
    
    ```bash
    # Through Claude Code
    Request the agent-template-wizard to create a new agent
    ```
  </Step>

  <Step title="Provide Domain Details">
    Specify:
    - Primary purpose and use case
    - Specific capabilities (3-5 concrete items)
    - Domain expertise area
    - Target user scenarios
  </Step>

  <Step title="Choose Category and Color">
    Select the appropriate category and color based on the agent's domain:
    - Development: blue
    - Operations: green  
    - Data/AI: yellow
    - Specialized: red
  </Step>

  <Step title="Define the 7 Steps">
    Work with the wizard to fill in all 7 methodology steps with domain-specific content.
  </Step>

  <Step title="Add Examples">
    Include 2 realistic example scenarios showing when and how the agent would be invoked.
  </Step>

  <Step title="Validate">
    Ensure all placeholders are replaced and the agent file is properly formatted.
  </Step>
</Steps>

<Warning>
All placeholders must be replaced. The validation system checks for remaining `{PLACEHOLDER}` markers and will fail if any are found.
</Warning>

## Agent vs Command

Understand when to use agents versus commands:

| Feature | Commands | Agents |
|---------|----------|--------|
| **Purpose** | Execute specific workflows | Provide domain expertise |
| **Structure** | 7 sections + XML config | 7-step methodology |
| **Location** | `.claude/commands/NN-phase/` | `.claude/agents/` |
| **Invocation** | `/command-name` | Automatic or manual |
| **Scope** | Task-specific | Domain-specific |
| **Examples** | `/audit-security`, `/test` | `rust-expert`, `performance-optimizer` |

## Real-World Agent Example

Here's the complete `performance-optimizer` agent showing the 7-step methodology:

```markdown
---
name: performance-optimizer
description: Use this agent when you need system-wide performance optimization, efficiency improvements, or holistic performance analysis across multiple domains.
color: green
---

You are an elite performance optimization specialist with deep expertise in 
systems thinking, efficiency analysis, resource optimization, and holistic 
performance improvement.

When optimizing performance, you will:

1. **Conduct Holistic Performance Analysis**: Analyze performance across 
   technical systems, business processes, and organizational workflows to 
   identify systemic bottlenecks and optimization opportunities.

2. **Apply Systems Thinking Approaches**: Understand interdependencies, 
   feedback loops, and emergent behaviors that affect overall system 
   performance and efficiency.

3. **Optimize Resource Utilization**: Maximize efficiency of human resources, 
   technical infrastructure, financial resources, and time allocation across 
   all system components.

4. **Design Performance Architecture**: Create integrated performance 
   frameworks that align technical performance with business objectives and 
   organizational capabilities.

5. **Implement Measurement and Monitoring**: Establish comprehensive 
   performance metrics, monitoring systems, and feedback mechanisms to track 
   optimization effectiveness.

6. **Execute Continuous Improvement**: Design performance improvement 
   processes that enable ongoing optimization and adaptation to changing 
   requirements and constraints.

7. **Balance Competing Objectives**: Optimize for multiple performance 
   dimensions including speed, quality, cost, scalability, and maintainability 
   while managing trade-offs effectively.

Your responses should be systematic, measurable, and sustainable. Always 
consider long-term performance implications and system-wide effects.
```

## Best Practices

<CardGroup cols={2}>
  <Card title="Focus and Clarity" icon="bullseye">
    Agents should have clear, specific domains. Avoid creating overly broad agents that try to cover too many areas.
  </Card>
  
  <Card title="Consistent Structure" icon="sitemap">
    Always follow the 7-step methodology. This ensures predictable, thorough responses across all agents.
  </Card>
  
  <Card title="Rich Examples" icon="code">
    Include realistic scenarios in the description that show when the agent should be invoked.
  </Card>
  
  <Card title="Color Coding" icon="palette">
    Use appropriate colors for visual categorization. This helps users quickly identify agent types.
  </Card>
</CardGroup>

# Safety System

ccprompts includes a comprehensive 3-layer safety system that uses Dagger containers to execute potentially dangerous commands in isolated environments, preventing accidental damage to your system.

## Overview

The safety system protects against dangerous operations through containerized execution, pattern detection, and rollback validation. Commands marked as "dangerous" execute in isolated Ubuntu 22.04 containers with resource limits and read-only project mounts.

<Note>
**Safety Validation Stats**: Currently achieving 65.7% safety validation pass rate across 517+ test cases with comprehensive audit logging.
</Note>

## Three-Layer Architecture

The safety system operates through three complementary layers:

<Steps>
  <Step title="Pattern Detection">
    Identifies dangerous commands automatically using regex patterns before execution.
  </Step>
  
  <Step title="Container Isolation">
    Executes dangerous commands in isolated Dagger containers with strict resource limits.
  </Step>
  
  <Step title="Rollback Validation">
    Verifies execution safety before allowing changes to persist to the host system.
  </Step>
</Steps>

## Layer 1: Pattern Detection

The pattern detection system uses `scripts/config/safety-patterns.js` to classify commands by danger level:

### Critical Patterns

Immediate security threats that always require containerization:

```javascript
CRITICAL_PATTERNS: [
  {
    pattern: /rm\s+-rf\s+[\/\$]/gi,
    severity: 'critical',
    message: 'Recursive file deletion with system paths',
    category: 'filesystem'
  },
  {
    pattern: /curl.*\|\s*(sh|bash)/gi,
    severity: 'critical',
    message: 'Pipe to shell from curl (remote code execution)',
    category: 'network'
  },
  {
    pattern: /wget.*\|\s*(sh|bash)/gi,
    severity: 'critical',
    message: 'Pipe to shell from wget (remote code execution)',
    category: 'network'
  },
  {
    pattern: /docker.*--privileged/gi,
    severity: 'critical',
    message: 'Docker privileged mode execution',
    category: 'container'
  }
]
```

### High-Risk Patterns

Significant security concerns requiring containerization:

```javascript
HIGH_RISK_PATTERNS: [
  {
    pattern: /sudo\s+(?!apt|npm|pip|yum|dnf)/gi,
    severity: 'high',
    message: 'Sudo execution with non-package managers',
    category: 'privilege'
  },
  {
    pattern: /chmod\s+[0-7]{3}/gi,
    severity: 'high',
    message: 'File permission changes',
    category: 'filesystem'
  },
  {
    pattern: /eval\s*\(/gi,
    severity: 'high',
    message: 'Dynamic code evaluation',
    category: 'execution'
  }
]
```

### Medium-Risk Patterns

Operations requiring caution but may not need containerization in all contexts:

```javascript
MEDIUM_RISK_PATTERNS: [
  {
    pattern: /(?:rm\s+-rf|delete\s+.*\/|destroy\s+.*\/)/gi,
    severity: 'medium',
    message: 'Potentially destructive file operations',
    category: 'filesystem',
    skipIfIncludes: ['example', 'placeholder', 'test', 'demo']
  },
  {
    pattern: /(?:curl|wget).*install|download.*\|\s*(?:sh|bash)/gi,
    severity: 'medium',
    message: 'Potentially unsafe software installation',
    category: 'network',
    skipIfIncludes: ['example', 'placeholder', 'test']
  }
]
```

<Accordion title="Pattern Categories">
Patterns are organized into categories for better analysis:

- **filesystem**: File operations, deletions, permission changes
- **network**: Remote downloads, curl/wget operations
- **privilege**: Sudo, elevated permissions
- **execution**: eval, exec, dynamic code
- **container**: Docker, containerization flags
</Accordion>

## Layer 2: Container Isolation

Dangerous commands execute in isolated Dagger containers using `scripts/safe-run.sh`:

### Container Configuration

```bash

# Safe command execution wrapper using Dagger containers

    "chmod -R"
    "chown -R"

    "wget.*|.*bash"

```

## Layer 3: Rollback Validation

The rollback validation layer uses `scripts/safety-validator.js` to verify execution safety:

### Safety Validation Process

```javascript
class SafetyValidator {
  /**
   * Validate command using Dagger container
   */
  async validateCommandInContainer(command, filename) {
    if (!this._daggerAvailable) {
      return {
        success: false,
        error: 'Dagger not available',
        containerValidated: false
      };
    }

    try {
      const startTime = Date.now();
      const safeRunScript = path.join(this.projectRoot, 'scripts', 'safe-run.sh');

      const result = execSync(
        `"${safeRunScript}" "${command}" --test`,
        {
          encoding: 'utf8',
          timeout: 10000,
          cwd: this.projectRoot
        }
      );

      const executionTime = Date.now() - startTime;
      this.safetyResults.containerTests++;

      return {
        success: true,
        output: result,
        executionTime,
        containerValidated: true,
        safetyLevel: this.determineSafetyLevel(command),
        recommendations: this.generateSafetyRecommendations(command)
      };
    } catch (error) {
      return {
        success: false,
        error: error.message,
        containerValidated: false
      };
    }
  }
}
```

### Safety Level Classification

Commands are classified into safety levels:

```javascript
determineSafetyLevel(command) {
  const criticalPatterns = ['rm -rf', 'curl.*|.*bash', 'sudo'];
  const highPatterns = ['chmod', 'docker.*--privileged', 'eval'];
  const mediumPatterns = ['npm install -g', 'systemctl', 'wget'];

  if (criticalPatterns.some(pattern => new RegExp(pattern, 'i').test(command))) {
    return 'critical';
  }
  if (highPatterns.some(pattern => new RegExp(pattern, 'i').test(command))) {
    return 'high';
  }
  if (mediumPatterns.some(pattern => new RegExp(pattern, 'i').test(command))) {
    return 'medium';
  }
  return 'low';
}
```

### Safety Recommendations

The system generates specific recommendations for dangerous operations:

```javascript
generateSafetyRecommendations(command) {
  const recommendations = [];

  if (/rm\s+-rf/i.test(command)) {
    recommendations.push('Consider using specific file paths instead of wildcards');
    recommendations.push('Always run in containers when dealing with file deletion');
  }

  if (/curl.*\|.*bash/i.test(command)) {
    recommendations.push('Download scripts to temporary files and inspect before execution');
    recommendations.push('Use signature verification when available');
  }

  if (/sudo/i.test(command)) {
    recommendations.push('Consider using rootless alternatives when possible');
    recommendations.push('Always validate the need for elevated privileges');
  }

  return recommendations;
}
```

## Performance Considerations

- **Container Startup**: ~500ms overhead for Dagger container initialization
- **Validation Time**: Currently ~1.8s for 70 commands
- **Resource Usage**: Configurable CPU/memory limits prevent exhaustion
- **Cleanup**: Automatic container cleanup prevents disk space issues

```bash Linux
curl -fsSL https://dl.dagger.io/dagger/install.sh | BIN_DIR=$HOME/.local/bin sh
```

```bash macOS
brew install dagger/tap/dagger
```

```bash Windows
winget install Dagger.Cli
```
</CodeGroup>

<Note>

</Note>

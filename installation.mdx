## Installation Methods

ccprompts offers two installation approaches: **plugin installation** for instant access across all projects, or **repository installation** for full customization and development.

## Method 1: Plugin Installation (Recommended)

Install ccprompts as a Claude Code plugin for instant access to all 70+ commands and 10 specialized agents in any project.

### Quick Plugin Installation

<Steps>
  <Step title="Add the Marketplace">
    Add the ccprompts marketplace in Claude Code:
    
    ```bash
    /plugin marketplace add ursisterbtw/ccprompts
    ```
  </Step>
  
  <Step title="Install the Plugin">
    Install ccprompts from the marketplace:
    
    ```bash
    /plugin install ccprompts@ursisterbtw
    ```
  </Step>
  
  <Step title="Restart Claude Code">
    Restart Claude Code to activate the plugin and load all commands
  </Step>
  
  <Step title="Verify Installation">
    Check that commands are available:
    
    ```bash
    /help
    ```
    
    You should see all 70+ ccprompts commands listed
  </Step>
</Steps>

<Tip>
After installation, try `/analyze-project` to get personalized recommendations for your codebase.
</Tip>

### Team-Wide Installation

For automatic installation across your entire team, add ccprompts to your project's `.claude/settings.json`:

```json .claude/settings.json
{
  "pluginMarketplaces": [
    "ursisterbtw/ccprompts"
  ],
  "plugins": [
    "ccprompts@ursisterbtw"
  ]
}
```

Team members who trust the repository folder will automatically get the plugin installed when they open the project.

<Note>
This requires team members to trust the repository folder in Claude Code settings.
</Note>

## Method 2: Repository Installation

Clone the repository for full customization, private extensions, or contribution development.

### Prerequisites

<AccordionGroup>
  <Accordion title="Bun (Required for validation)" icon="terminal">
    Install Bun for running validation and tests:
    
    **Linux/macOS:**
    ```bash
    curl -fsSL https://bun.sh/install | bash
    ```
    
    **Windows:**
    Use WSL with the script above or the official installer from [bun.sh](https://bun.sh)
    
    Verify installation:
    ```bash
    bun --version
    ```
  </Accordion>
  
  <Accordion title="Dagger (Recommended for safety)" icon="docker">
    Install Dagger for containerized command execution (optional but recommended):
    
    **Linux:**
    ```bash
    curl -fsSL https://dl.dagger.io/dagger/install.sh | BIN_DIR=$HOME/.local/bin sh
    ```
    
    **macOS:**
    ```bash
    brew install dagger/tap/dagger
    ```
    
    **Windows:**
    ```powershell
    winget install Dagger.Cli
    ```
  </Accordion>
</AccordionGroup>

### Repository Clone and Setup

<CodeGroup>

```bash Standard Installation
# Clone the repository
git clone https://github.com/ursisterbtw/ccprompts.git
cd ccprompts

# Install dependencies
bun install

# Validate all commands
bun run validate
```

```fish Fish Shell (One-liner)
# Clone, navigate, install, and validate in one command
git clone https://github.com/ursisterbtw/ccprompts.git; and cd ccprompts; and bun i; and bun run validate
```

```bash Local Development Setup
# Clone the repository
git clone https://github.com/ursisterbtw/ccprompts.git
cd ccprompts

# Install dependencies
bun install

# Run full CI pipeline (validate + lint + link-check)
bun run ci

# Add as local marketplace
/plugin marketplace add /path/to/ccprompts

# Install the plugin
/plugin install ccprompts@ccprompts
```

</CodeGroup>

## Configuration

### Plugin Permissions

The plugin requires these permissions (automatically configured):

```json Plugin Permissions
{
  "permissions": {
    "allow": [
      "Bash(git:*)",
      "Bash(npm:*)",
      "Bash(pip:*)",
      "Bash(cargo:*)",
      "Read",
      "Write",
      "Edit",
      "Glob",
      "Grep"
    ],
    "deny": [
      "Bash(rm -rf:*)",
      "Bash(sudo:*)"
    ]
  }
}
```

<Warning>
Dangerous commands like `rm -rf` and `sudo` are automatically denied and must be executed through the Dagger safety system.
</Warning>

### Environment Variables

Optional environment configuration for enhanced functionality:

```json Environment Configuration
{
  "env": {
    "CLAUDE_CODE_PROJECT_TYPE": "command-ecosystem",
    "CLAUDE_CODE_ENABLE_MCP": "1"
  }
}
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Commands Not Available After Installation">
    **Solution:**
    
    1. **Restart Claude Code** - Required after plugin installation
    2. **Check Installation** - Run `/plugin` to verify ccprompts is installed
    3. **Verify Marketplace** - Ensure marketplace was added correctly:
       ```bash
       /plugin marketplace list
       ```
  </Accordion>
  
  <Accordion title="Permission Errors">
    **Solution:**
    
    1. Check your `.claude/settings.json` permissions configuration
    2. Trust the repository folder if using team configuration
    3. Review the command's required permissions in the command documentation
    4. Dangerous commands require Dagger safety system
  </Accordion>
  
  <Accordion title="Validation Failures (Repository)">
    **Solution:**
    
    1. **Check Node.js version** - Requires Node.js >= 18.0.0, < 23.0.0
       ```bash
       node --version
       ```
    2. **Reinstall dependencies**
       ```bash
       bun install --frozen-lockfile
       ```
    3. **Check expected command count** - Should have exactly 70 commands
       ```bash
       EXPECTED_COMMAND_COUNT=70 node scripts/validate-commands.js
       ```
  </Accordion>
  
  <Accordion title="Symlink Issues (Local Development)">
    **Solution for Windows:**
    
    Use directory junctions instead of symlinks:
    ```powershell
    mklink /J commands .claude\commands
    mklink /J agents .claude\agents
    ```
    
    **Alternative (not recommended for development):**
    ```bash
    cp -r .claude/commands commands
    cp -r .claude/agents agents
    ```
  </Accordion>
  
  <Accordion title="Dagger Not Available">
    **Solution:**
    
    Dagger is optional but recommended. If Dagger is not installed:
    
    1. Commands will still work but dangerous operations won't be containerized
    2. Install Dagger for full safety system support (see Prerequisites above)
    3. The safety system provides 65.7% validation rate across 517+ test cases
  </Accordion>
</AccordionGroup>

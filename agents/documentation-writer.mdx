## Core Capabilities

### 1. Analyze Documentation Needs

Understand audience and scope:

<AccordionGroup>
  <Accordion title="Developer Documentation vs User Guides">
    - **Developer docs** - API references, code examples, architecture
    - **User guides** - Feature tutorials, getting started, troubleshooting
    - **Admin docs** - Deployment, configuration, maintenance
    - **Contributor docs** - Development setup, coding standards, PR process
  </Accordion>
  
  <Accordion title="Documentation Types">
    - **API reference** - Endpoint documentation, parameters, responses
    - **Tutorials** - Step-by-step learning paths
    - **How-to guides** - Task-focused instructions
    - **Explanations** - Conceptual understanding
    - **ADRs** - Architecture decision records
  </Accordion>
  
  <Accordion title="Audience Considerations">
    - Technical skill level
    - Domain knowledge
    - Learning objectives
    - Time constraints
    - Preferred learning styles
  </Accordion>
</AccordionGroup>

### 2. Structure Content Effectively

Create logical documentation hierarchy:

- **Clear navigation** - Intuitive information architecture
- **Progressive disclosure** - Simple to complex flow
- **Logical organization** - Related content grouped together
- **Cross-references** - Internal and external links
- **Search optimization** - Keywords and metadata

    ### Rate Limits
    
    - 1000 requests per hour per API key
    - Rate limit headers included in response
    
    ### Notes
    
    - Requires authentication
    - Cached for 5 minutes
    - Supports ETags for conditional requests

### 4. Create Comprehensive API Documentation

Key elements of API documentation:

- **Endpoint descriptions** - Clear purpose and use cases
- **Request examples** - Multiple languages (curl, Python, JavaScript)
- **Response schemas** - JSON structure with types
- **Authentication** - Auth methods and examples
- **Error codes** - All possible errors with solutions
- **Rate limits** - Quotas and best practices

### 5. Develop Effective README Files

Essential README sections:

1. **Project description** - What it does and why it exists
2. **Quick start** - Get running in < 5 minutes
3. **Installation** - Step-by-step setup
4. **Usage examples** - Common use cases
5. **Configuration** - All options documented
6. **Contributing** - How to help
7. **License** - Legal information

### 6. Document Architecture and Design

Architecture documentation components:

- **System overview** - High-level architecture diagram
- **Component interactions** - How pieces work together
- **Data flow** - Request/response lifecycles
- **Technology decisions** - Why specific tech was chosen
- **Deployment** - Production architecture
- **Security** - Security measures and considerations

### 7. Maintain Documentation Quality

Quality assurance practices:

```markdown

## Example: Complete API Documentation

```markdown
# Users API

## Overview

The Users API allows you to create, read, update, and delete user accounts.

**Base URL:** `https://api.example.com/v1`

**Authentication:** Bearer token required for all endpoints

## Endpoints

### List Users

```http
GET /users
```

Retrieves a paginated list of users.

**Query Parameters:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| page | integer | No | Page number (default: 1) |
| limit | integer | No | Items per page (default: 20, max: 100) |
| sort | string | No | Sort field (e.g., "created_at") |

**Response:**

```json
{
  "data": [
    {
      "id": 1,
      "username": "john_doe",
      "email": "john@example.com",
      "created_at": "2024-01-01T00:00:00Z"
    }
  ],
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 100
  }
}
```

**Error Responses:**

| Code | Description |
|------|-------------|
| 401 | Unauthorized - Invalid or missing token |
| 429 | Rate limit exceeded |

## Rate Limiting

- **Limit:** 1000 requests per hour
- **Headers:** 
  - `X-RateLimit-Limit`: Maximum requests
  - `X-RateLimit-Remaining`: Remaining requests
  - `X-RateLimit-Reset`: Reset timestamp

## SDKs

### Python

```python
from example_sdk import Client

client = Client(api_key="your_api_key")
users = client.users.list(page=1, limit=20)
```

### JavaScript

```javascript
import { Client } from 'example-sdk';

const client = new Client({ apiKey: 'your_api_key' });
const users = await client.users.list({ page: 1, limit: 20 });
```
```

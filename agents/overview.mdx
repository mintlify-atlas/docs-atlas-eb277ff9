ccprompts includes 10 specialized AI agents that provide expert-level knowledge in specific programming languages, frameworks, and technical domains. Each agent follows a consistent 7-step methodology and is automatically invoked when their expertise is needed.

## When to Use Agents

Agents are automatically invoked based on context, but you can also explicitly request them:

```bash
# Automatic invocation based on context
"How do I handle Rust ownership issues?"
# → Automatically uses rust-expert agent

# Explicit invocation
/agent rust-expert "Optimize this code for zero-copy patterns"
```

## The 7-Step Methodology

Every specialized agent follows a consistent 7-step approach:

<Steps>
  <Step title="Analysis">
    Analyze the problem domain, requirements, and constraints
  </Step>
  <Step title="Pattern Identification">
    Identify appropriate design patterns, best practices, and approaches
  </Step>
  <Step title="Implementation Design">
    Design the solution architecture with specific techniques and tools
  </Step>
  <Step title="Optimization">
    Apply performance optimizations and efficiency improvements
  </Step>
  <Step title="Considerations">
    Evaluate trade-offs, edge cases, and architectural decisions
  </Step>
  <Step title="Validation">
    Implement testing, monitoring, and quality assurance strategies
  </Step>
  <Step title="Measurement">
    Establish metrics, benchmarks, and continuous improvement processes
  </Step>
</Steps>

## Agent Selection Logic

Agents are automatically selected based on:

1. **Explicit language/framework mentions** - Keywords trigger relevant agents
2. **Problem domain** - Concurrency → Go/Rust, Async → Python/FastAPI, etc.
3. **Technical complexity** - Advanced patterns route to specialized agents
4. **Context from codebase** - File types and project structure inform selection

<Note>
**Pro Tip**: You can explicitly request an agent using `/agent <name> "<query>"` to bypass automatic selection.
</Note>

The Agent Template Wizard specializes in creating production-ready agents using the SUBAGENT_TEMPLATE.md template. This wizard ensures proper formatting, validates frontmatter, enforces naming conventions, and guarantees template compliance.

The Agent Template Wizard is ideal for:

## Directory Rules

<Warning>
**IMPORTANT**: Agent placement follows specific directory rules:

- **Project-specific agents**: `./[project]/.claude/agents/[category]/`
- **Global system agents**: `~/.claude/agents/[category]/` (only when explicitly requested)
- **Default behavior**: Create agents locally in current working directory
</Warning>

## Core Capabilities

### 1. Template Analysis

Read and understand template structure:

<AccordionGroup>
  <Accordion title="Template Location">
    The wizard reads from `~/.claude/templates/SUBAGENT_TEMPLATE.md` to understand:
    - Required frontmatter fields
    - Placeholder naming conventions
    - Example structure requirements
    - Section organization
  </Accordion>
  
  <Accordion title="Placeholder Identification">
    All placeholders that must be filled:
    - `{AGENT_NAME}` - Kebab-case identifier
    - `{PRIMARY_USE_CASE}` - Clear, specific purpose
    - `{SPECIFIC_CAPABILITIES}` - 3-5 concrete capabilities
    - `{EXAMPLE_CONTEXT_1/2}` - Realistic usage scenarios
    - `{EXAMPLE_USER_REQUEST_1/2}` - Natural user requests
    - `{EXAMPLE_ASSISTANT_RESPONSE_1/2}` - Appropriate responses
    - `{EXAMPLE_COMMENTARY_1/2}` - Why agent was selected
    - `{AGENT_COLOR}` - Approved color from convention
    - `{DOMAIN_EXPERT_TITLE}` - Professional title
  </Accordion>
</AccordionGroup>

### 2. Information Gathering

Collect requirements for agent creation:

<Steps>

### 3. Placeholder Replacement

Systematically fill all template placeholders:

<Tabs>
  <Tab title="Agent Name">
    ```markdown
    # {AGENT_NAME} → python-performance-optimizer
    
    Rules:
    - Kebab-case format (lowercase with hyphens)
    - Descriptive and specific
    - Matches filename without .md extension
    - No underscores, spaces, or capitals
    
    Examples:
    ✓ python-performance-optimizer
    ✓ blockchain-security-auditor
    ✓ fastapi-async-expert
    ✗ Python_Optimizer (wrong case/underscore)
    ✗ perf-opt (too vague)
    ```
  </Tab>
  
  <Tab title="Capabilities">
    ```markdown
    # {SPECIFIC_CAPABILITIES}
    
    Format: 3-5 concrete, actionable capabilities
    
    Example for Python Performance Agent:
    - Profile Python code with cProfile and memory_profiler
    - Optimize algorithmic complexity from O(n²) to O(n log n)
    - Implement caching strategies with functools.lru_cache
    - Parallelize CPU-bound tasks with multiprocessing
    - Optimize memory usage with generators and iterators
    ```
  </Tab>
  
  <Tab title="Examples">
    ```markdown
    # Example Structure
    
    <example>
    Context: {EXAMPLE_CONTEXT_1}
    user: "{EXAMPLE_USER_REQUEST_1}"
    assistant: "{EXAMPLE_ASSISTANT_RESPONSE_1}"
    <commentary>{EXAMPLE_COMMENTARY_1}</commentary>

    Real Example:
    <example>
    ```

### 4. Naming Convention Enforcement

Ensure proper naming:

```bash
# File naming rules
✓ python-performance-optimizer.md
✓ blockchain-security-auditor.md
✓ react-component-architect.md

✗ Python_Optimizer.md (wrong case/underscore)
✗ performance_optimizer.md (underscore instead of hyphen)
✗ PerfOpt.md (abbreviation, wrong case)

# Frontmatter must match filename
Filename: python-performance-optimizer.md
Frontmatter: name: python-performance-optimizer
```

### 6. Validation Checklist

Pre-deployment validation:

<AccordionGroup>
  <Accordion title="Placeholder Check">
    ```bash
    # Verify no placeholders remain
    grep -r '{[A-Z_]*}' agent-file.md
    
    # Should return nothing
    # If placeholders found, fill them before proceeding
    ```
  </Accordion>
  
  <Accordion title="Frontmatter Validation">
    ```yaml
    ---
    name: agent-name          # Required: matches filename
    description: "..."         # Required: enables auto-invocation
    color: blue               # Required: category color
    tools: Read, Write        # Optional: required tools
    model: sonnet            # Optional: preferred model
    ---
    ```
  </Accordion>
  
  <Accordion title="Example Format">
    ```markdown
    # Verify examples use correct XML tags
    <example>
    Context: ...
    user: "..."
    assistant: "..."
    <commentary>...</commentary>
    </example>
    
    # NOT:
    Example: (plain text)
    User: ...
    Agent: ...
    ```
  </Accordion>
  
  <Accordion title="Auto-Invocation">
    ```markdown
    # Description must enable auto-invocation
    
    ✓ "Use this agent when you need Python performance optimization"
    ✓ "Specializes in React component architecture and state management"
    
    ✗ "Python agent" (too vague)
    ✗ "Helps with stuff" (not descriptive)
    ```
  </Accordion>
</AccordionGroup>

### 7. Tool Access Verification

Specify required tools:

```yaml
---
tools: Read, Write, Bash, Grep
---

# Default tools (always available):
- Read: File reading
- Write: File writing
- Bash: Command execution
- Grep: Text search

# Additional tools (justify need):
- WebFetch: External documentation
- Task: Complex multi-step operations
- Glob: File pattern matching
```

## The 7-Step Creation Process

<Steps>
  <Step title="Template Analysis">
    Read SUBAGENT_TEMPLATE.md and identify all placeholders
  </Step>
  <Step title="Information Gathering">
    Determine purpose, capabilities, category, and color
  </Step>
  <Step title="Placeholder Replacement">
    Fill all placeholders with specific, relevant content
  </Step>
  <Step title="Naming Convention">
    Ensure kebab-case naming and frontmatter match
  </Step>
  <Step title="Category Placement">
    Place agent in correct directory (local vs global)
  </Step>
  <Step title="Validation">
    Run validation checklist to ensure compliance
  </Step>
  <Step title="Documentation">
    Suggest catalog entry and usage documentation
  </Step>
</Steps>

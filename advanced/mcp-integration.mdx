The Model Context Protocol (MCP) allows you to extend Claude Code with powerful integrations to external services, databases, APIs, and custom tools. ccprompts provides comprehensive MCP integration commands for discovering, configuring, and managing MCP servers.

## What is MCP?

MCP (Model Context Protocol) is an open protocol that enables Claude to connect to external data sources and tools through standardized server implementations.

## MCP Commands in ccprompts

ccprompts provides four main MCP commands:

### Command Overview

<Tabs>
  <Tab title="/mcp">
    **Basic MCP setup and configuration**
    
    ```bash
    /mcp [action] [integration]
    ```
    
    **Actions:**
    - `setup` - Configure new MCP server
    - `test` - Test MCP server connection
    - `configure` - Modify server settings
    - `troubleshoot` - Debug integration issues
    - `develop` - Create custom MCP server
    
    **Examples:**
    ```bash
    /mcp setup github
    /mcp test database
    /mcp configure monitoring
    /mcp develop custom
    ```
  </Tab>
  
  <Tab title="/mcp-discover">
    **Discover and install MCP servers**
    
    ```bash
    /mcp-discover [category] [--install] [--evaluate]
    ```
    
    **Categories:**
    - `development` - Development tools
    - `database` - Database connections
    - `api` - API integrations
    - `ai` - AI/ML services
    - `productivity` - Project management
    
    **Examples:**
    ```bash
    /mcp-discover database --install
    /mcp-discover --interactive
    /mcp-discover development --stack typescript
    ```
  </Tab>
  
  <Tab title="/mcp-configure">


    **Features:**

    **Examples:**
    ```bash

    /mcp-configure redis --security strict
    ```

  </Tab>

    **Capabilities:**
    - Custom tools and resources

    ```bash

    /mcp test postgres
    ```
    
    **Test results:**

    ```
    ✓ Schema access verified
    ✓ Query execution working
    ✓ Security restrictions applied
    ```

## Configuring MCP Servers

### Basic Configuration

MCP servers are configured in `.claude/mcp-config.json`:

<CodeGroup>
```json Simple Configuration
{
  "mcp_servers": {
    "postgres": {
      "server": "@modelcontextprotocol/server-postgres",
      "config": {
        "connection_string": "${DATABASE_URL}",
        "read_only": true
      }
    }
  }
}
```

```json Multi-Environment
{
  "mcp_servers": {
    "postgres": {
      "server": "@modelcontextprotocol/server-postgres",
      "environments": {
        "development": {
          "connection_string": "${DEV_DATABASE_URL}",
          "read_only": false
        },
        "production": {
          "connection_string": "${PROD_DATABASE_URL}",
          "read_only": true
        }
      }
    }
  }
}
```

```json Advanced Security
{
  "mcp_servers": {
    "postgres": {
      "server": "@modelcontextprotocol/server-postgres",
      "config": {
        "connection_string": "${DATABASE_URL}",
        "read_only": true,
        "max_connections": 5,
        "timeout": 30000
      },
      "security": {
        "sandbox": true,
        "network_access": "limited",
        "allowed_operations": ["SELECT", "EXPLAIN"],
        "denied_tables": ["users", "passwords"]
      },
      "monitoring": {
        "log_queries": true,
        "log_level": "info",
        "alert_on_errors": true
      }
    }
  }
}
```
</CodeGroup>

### Environment Variables

Store sensitive data in environment variables:

```bash
# .env file
DATABASE_URL=postgres://user:pass@localhost:5432/mydb
GITHUB_TOKEN=ghp_xxxxxxxxxxxxxxxxxxxx
SLACK_WEBHOOK=https://hooks.slack.com/services/xxx
REDIS_URL=redis://localhost:6379
```

<Warning>
Never commit `.env` files or hardcode credentials. Always use environment variables for sensitive configuration.
</Warning>

### Custom MCP Server Development

Create project-specific MCP servers:

<Steps>
  <Step title="Initialize Server">
    ```bash
    /mcp develop custom --name project-api-server
    ```
  </Step>
  
  <Step title="Define Tools">
    ```typescript
    // server.ts
    import { MCPServer } from '@modelcontextprotocol/sdk';
    
    const server = new MCPServer({
      name: 'project-api-server',
      version: '1.0.0'
    });
    
    // Custom tool: Deploy to staging
    server.addTool({
      name: 'deploy_staging',
      description: 'Deploy current branch to staging environment',
      parameters: {
        branch: { type: 'string', required: true },
        runTests: { type: 'boolean', default: true }
      },
      handler: async (params) => {
        // Deployment logic
        return { success: true, url: 'https://staging.example.com' };
      }
    });
    ```
  </Step>
  
  <Step title="Register Server">
    ```bash
    /mcp-configure project-api-server --local ./mcp-server
    ```
  </Step>
  
  <Step title="Use Custom Tools">
    ```
    Claude: "Deploy current branch to staging"
    → Uses custom deploy_staging tool
    ✓ Deployed to https://staging.example.com
    ```
  </Step>
</Steps>

## Best Practices

### Security

<Card title="Security Checklist" icon="shield-check">
  **Configuration:**
  - ✓ Use environment variables for credentials
  - ✓ Enable read-only mode when possible
  - ✓ Limit network access
  - ✓ Use sandbox mode for untrusted operations
  - ✓ Restrict allowed operations
  
  **Monitoring:**
  - ✓ Enable audit logging
  - ✓ Monitor server usage
  - ✓ Alert on suspicious activity
  - ✓ Regular security audits
  
  **Access Control:**
  - ✓ Principle of least privilege
  - ✓ Separate dev/prod credentials
  - ✓ Rotate credentials regularly
  - ✓ Review access logs
</Card>

### Performance

<CardGroup cols={2}>
  <Card title="Connection Pooling" icon="arrows-split-up-and-left">

## Troubleshooting

### Common Issues

<AccordionGroup>
  <Accordion title="Connection Failures">
    **Symptoms:**
    - "Failed to connect to MCP server"
    - "Connection timeout"
    
    **Solutions:**
    ```bash
    # Verify server is running
    /mcp test postgres
    
    # Check configuration
    /mcp-configure postgres --verify
    
    # Review logs
    /mcp troubleshoot postgres --logs
    
    # Test network connectivity
    /mcp troubleshoot postgres --network
    ```
  </Accordion>
  
  <Accordion title="Authentication Errors">
    **Symptoms:**
    - "Authentication failed"
    - "Invalid credentials"
    
    **Solutions:**
    ```bash
    # Verify environment variables
    echo $DATABASE_URL
    echo $GITHUB_TOKEN
    
    # Reconfigure authentication
    /mcp-configure postgres --auth-reset
    
    # Test with explicit credentials
    /mcp test postgres --connection-string "postgres://..."
    ```
  </Accordion>
  
  <Accordion title="Performance Issues">
    **Symptoms:**
    - Slow query execution
    - Timeout errors
    
    **Solutions:**
    ```bash
    # Check server metrics
    /mcp monitor postgres
    
    # Analyze slow queries
    /mcp troubleshoot postgres --slow-queries
    
    # Optimize configuration
    /mcp-configure postgres --optimize
    ```
  </Accordion>
</AccordionGroup>

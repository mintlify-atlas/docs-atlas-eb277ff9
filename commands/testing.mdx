## Overview

Phase 04 commands help you build robust test suites and systematically debug issues across your development workflow.

### Key Capabilities

**Test Generation**
- Automatic test creation for untested code
- Unit, integration, and end-to-end test suites
- Framework-specific test generation (Jest, Pytest, etc.)
- Coverage analysis and gap identification
- Property-based and mutation testing

**Test Automation**
- Continuous testing in CI/CD
- Pre-commit test hooks
- Test performance optimization
- Regression detection
- Test maintenance and refactoring

**Systematic Debugging**
- Urgency-based response protocols
- Error pattern recognition
- AI-powered analysis
- Step-by-step debugging workflows
- Root cause analysis

**Troubleshooting Support**
- Production issue diagnosis
- Performance investigation
- Environment debugging
- Integration troubleshooting
- Knowledge capture and sharing

## Use Cases

### Generate Missing Tests

```bash
/test "missing coverage" "unit"
```

Automatically create unit tests for untested code paths and improve coverage.

### API Integration Testing

```bash
/test "api/users" "integration"
```

Generate comprehensive integration tests for API endpoints with mocking.

### Critical Production Error

```bash
/troubleshoot error production critical
```

Immediate triage and resolution guidance for critical production issues.

### Performance Investigation

```bash
/troubleshoot performance development medium
```

Systematic analysis and optimization of slow performance issues.

## Example Workflows

### Test-Driven Development

1. **Generate Tests**: `/test "new-feature" "unit"`
2. **Run Tests**: Execute test suite (should fail)
3. **Implement**: Write implementation code
4. **Validate**: Tests should pass
5. **Refactor**: Improve code with test safety net

### Bug Investigation and Fix

1. **Troubleshoot**: `/troubleshoot error development high`
2. **Reproduce**: Create failing test case
3. **Debug**: Systematic investigation
4. **Fix**: Implement solution
5. **Validate**: Verify fix with tests
6. **Document**: Update knowledge base

### Pre-Release Quality Assurance

1. **Unit Tests**: `/test "missing coverage" "unit"`
2. **Integration**: `/test "api/*" "integration"`
3. **End-to-End**: `/test "user-flows" "e2e"`
4. **Performance**: `/test "critical-paths" "performance"`
5. **Security**: `/test "auth-system" "security"`

## Best Practices

1. **Test Early**: Write tests alongside or before implementation code
2. **Maintain Tests**: Keep tests clean and up-to-date with code changes
3. **Coverage Goals**: Aim for 80%+ coverage on critical code
4. **Fast Tests**: Keep unit tests fast (&lt;1 second each)
5. **Isolated Tests**: Ensure tests don't depend on each other
6. **Clear Assertions**: Make test failures easy to understand
7. **Document Issues**: Capture troubleshooting knowledge for team
8. **Automate**: Integrate testing into CI/CD pipeline

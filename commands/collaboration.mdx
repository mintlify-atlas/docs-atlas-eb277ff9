Phase 06 commands enhance team productivity through automated reviews, monitoring, and systematic debt management.

## Use Cases

### Automated Code Review

```bash
/code-review PR-123 comprehensive
```

Comprehensive automated review of pull request with security, quality, and best practice feedback.

### Daily Standup Reports

```bash
/daily-standup team yesterday
```

Generate automated standup reports from Git activity, PRs, and issue tracking.

### Production Monitoring Setup

```bash
/monitor production full-stack
```

Complete monitoring and alerting setup for production applications.

### Technical Debt Assessment

```bash
/tech-debt assess full-codebase
```

Identify and prioritize technical debt across the entire codebase.

## Example Workflows

### Pull Request Review Process

1. **Automated Review**: `/code-review PR-456 comprehensive`
2. **Address Issues**: Fix identified problems
3. **Re-Review**: Automated validation of fixes
4. **Approval**: Merge when all checks pass
5. **Monitor**: Track deployment impact

### Daily Team Coordination

1. **Standup Report**: `/daily-standup team yesterday`
2. **Review Blockers**: Identify impediments
3. **Plan Today**: Set daily priorities
4. **Track Progress**: Monitor work items
5. **End-of-Day**: Update status

### Production Health Management

1. **Setup Monitoring**: `/monitor production full-stack`
2. **Configure Alerts**: Set threshold and escalation
3. **Dashboard**: Create team visibility
4. **Incident Response**: Automated alerts trigger response
5. **Post-Mortem**: Learn and improve

### Technical Debt Reduction

1. **Assess**: `/tech-debt assess full-codebase`
2. **Prioritize**: Rank by impact and effort
3. **Plan**: Create sprint-sized work items
4. **Execute**: Systematic debt reduction
5. **Track**: Monitor debt reduction progress

## Technical Debt Categories

### Code Debt

**Issues**:
- Code duplication
- Complex functions
- Poor naming
- Missing documentation
- Outdated patterns

**Impact**: Maintainability, developer productivity

### Architecture Debt

**Issues**:
- Tight coupling
- Missing abstractions
- Monolithic structures
- Scalability limitations
- Performance bottlenecks

**Impact**: Scalability, performance, extensibility

### Test Debt

**Issues**:
- Low test coverage
- Flaky tests
- Missing integration tests
- Outdated test data
- Poor test organization

**Impact**: Quality, confidence, velocity

### Documentation Debt

**Issues**:
- Outdated documentation
- Missing API docs
- Incomplete runbooks
- No architecture diagrams
- Missing decision records

**Impact**: Onboarding, knowledge sharing, maintenance

### Dependency Debt

**Issues**:
- Outdated dependencies
- Security vulnerabilities
- License incompatibilities
- Unused dependencies
- Multiple versions

**Impact**: Security, compatibility, maintenance

## Debt Prioritization

### High Priority

**Characteristics**:
- Security vulnerabilities
- Critical performance issues
- Blocking new features
- High developer friction
- Compliance requirements

**Timeline**: Address within current sprint

### Medium Priority

**Characteristics**:
- Moderate performance impact
- Affecting team velocity
- Moderate complexity
- Growing technical burden
- Quality degradation

**Timeline**: Address within 1-2 sprints

### Low Priority

**Characteristics**:
- Minor annoyances
- Low impact on velocity
- Easy to work around
- Cosmetic issues
- Future-proofing

**Timeline**: Address when capacity allows

## Overview

Phase 03 commands provide comprehensive security coverage from initial auditing through compliance automation, system hardening, and emergency incident response.

### Key Capabilities

**Security Auditing**
- OWASP Top 10 vulnerability scanning
- Code quality assessment
- Performance bottleneck identification
- Compliance gap analysis
- Dependency security review
- License health and compatibility

**Compliance Automation**
- Evidence collection for SOC2, GDPR, HIPAA, PCI-DSS
- Continuous controls monitoring
- Automated policy document generation
- Real-time compliance drift detection
- Audit trail generation

**System Hardening**
- Application security implementation
- Infrastructure security configuration
- Network security and access controls
- Container and cloud hardening
- Security monitoring setup

**Incident Response**
- Automated incident detection and classification
- Structured response workflows
- Real-time monitoring and status tracking
- Root cause analysis
- Post-incident review and improvement

## Example Workflows

### Pre-Production Security Review

1. **Audit**: `/audit-security full-codebase thorough`
2. **Compliance**: `/comply soc2 basic`
3. **Harden**: `/harden enterprise`
4. **Validate**: Re-run audit to verify improvements
5. **Document**: Generate security documentation

### Security Incident Handling

1. **Detect**: Automated alerts trigger incident
2. **Respond**: `/incident-response high security production`
3. **Contain**: Isolate affected systems
4. **Investigate**: Root cause analysis
5. **Remediate**: Apply fixes and harden
6. **Review**: Post-incident analysis

### Compliance Audit Preparation

1. **Gap Analysis**: `/comply hipaa thorough`
2. **Evidence Collection**: Automated artifact gathering
3. **Policy Generation**: Create required documentation
4. **Continuous Monitoring**: Real-time compliance tracking
5. **Audit Support**: Integration with audit portals

## Security Audit Levels

### Surface Level

Quick security scan covering:
- Critical vulnerabilities only
- High-risk dependencies
- Common misconfigurations
- Basic OWASP issues

**Duration**: 15-30 minutes
**Use Case**: Rapid feedback during development

### Thorough Level

Comprehensive analysis including:
- Full OWASP Top 10 coverage
- Dependency vulnerability analysis
- Code quality assessment
- Security best practices review
- Performance bottleneck identification

**Duration**: 1-2 hours
**Use Case**: Pre-release security validation

### Paranoid Level

Maximum security scrutiny with:
- Deep code analysis
- Advanced threat modeling
- Cryptographic review
- Complete dependency audit
- Compliance verification
- Penetration testing preparation

**Duration**: 3-5 hours
**Use Case**: High-security environments, regulated industries

## Incident Severity Levels

### Critical (SEV-1)

- **Impact**: Complete service outage or critical functionality unavailable
- **Response Time**: Immediate (within 15 minutes)
- **Communication**: Real-time updates every 30 minutes
- **Escalation**: Automatic to leadership

### High (SEV-2)

- **Impact**: Major functionality degraded or limited outage
- **Response Time**: Within 1 hour
- **Communication**: Updates every 2 hours
- **Escalation**: Team lead and on-call engineer

### Medium (SEV-3)

- **Impact**: Minor functionality issues or performance degradation
- **Response Time**: Within 4 hours (business hours)
- **Communication**: Daily updates until resolved
- **Escalation**: Assigned team member

### Low (SEV-4)

- **Impact**: Cosmetic issues or non-critical functionality
- **Response Time**: Within 5 business days
- **Communication**: Weekly updates in team meetings
- **Escalation**: Normal prioritization

## Incident Types

### System Outage

Complete or partial service unavailability:
- Immediate triage and assessment
- Emergency response procedures
- Rollback and recovery strategies
- Communication plan activation

### Security Incident

Security breach or compromise:
- System isolation and evidence preservation
- Forensic data collection
- Scope assessment and containment
- Legal and regulatory notification

### Performance Degradation

Slow response times or reduced capacity:
- Performance monitoring and profiling
- Resource scaling and optimization
- Bottleneck identification
- Gradual remediation

### Data Incident

Data loss, corruption, or exposure:
- Backup restoration procedures
- Data integrity verification
- Impact assessment
- Compliance notification

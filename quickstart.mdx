## Get Started in Three Steps

Install ccprompts, analyze your project, and start using production-ready commands in minutes.

<Steps>
  <Step title="Install ccprompts">
    Install as a plugin for instant access:
    
    ```bash
    /plugin marketplace add ursisterbtw/ccprompts
    /plugin install ccprompts@ursisterbtw
    ```
    
    Then **restart Claude Code** to activate all commands.
  </Step>
  
  <Step title="Analyze Your Project">
    Get personalized recommendations based on your codebase:
    
    ```bash
    /analyze-project
    ```
    
    This will detect your project type, technology stack, and suggest relevant commands.
  </Step>
  
  <Step title="Try Your First Command">
    Based on the analysis, try a recommended command. For example:
    
    ```bash
    /setup-ci github professional
    ```
  </Step>
</Steps>

<Tip>
Use `/help` at any time to see all 70+ available commands organized by development phase.
</Tip>

## Your First Commands

Here are the most useful commands to get started with ccprompts:

### Project Analysis

Understand your codebase and get personalized recommendations:

```bash
/analyze-project
```

**What it does:**
- Auto-detects project type, technology stack, and architecture
- Identifies development maturity level and gaps
- Suggests relevant commands based on project characteristics
- Provides timeline estimates for improvements
- Creates a personalized development roadmap

<Note>
The project analyzer is intelligent - it will recommend different commands for a legacy monolith versus a new startup project.
</Note>

### Security Audit

Run a comprehensive security analysis of your codebase:

<CodeGroup>

```bash Quick Security Scan
/audit-security full-codebase thorough
```

```bash Paranoid Mode (Maximum Depth)
/audit-security full-codebase paranoid
```

```bash Compliance-Focused
/compliance soc2 --enforce-automatically
```

</CodeGroup>

**What it does:**
- Scans for common vulnerabilities and security anti-patterns
- Checks dependency security and license compliance
- Analyzes authentication and authorization patterns
- Provides actionable remediation steps
- Generates security scorecard with priorities

### CI/CD Setup

Configure production-ready continuous integration:

<CodeGroup>

```bash GitHub Actions
/setup-ci github professional
```

```bash GitLab CI
/setup-ci gitlab enterprise
```

```bash Custom Pipeline
/setup-ci custom --platform jenkins
```

</CodeGroup>

**What it does:**
- Creates CI/CD configuration files for your platform
- Sets up automated testing, linting, and security scanning
- Configures deployment workflows
- Includes best practices for caching and optimization
- Adds status badges and notifications

### MCP Server Discovery

Extend Claude's capabilities with Model Context Protocol servers:

```bash
/mcp-discover database --install
```

**What it does:**
- Searches through 1000+ available MCP servers
- Evaluates server capabilities and security
- Automates installation and configuration
- Integrates seamlessly with existing workflows
- Enables database connections, API integrations, and more

<Tip>
Try `/mcp-discover development --stack typescript` to install a full development stack including ESLint, Prettier, TypeScript, and Jest servers.
</Tip>

## Working with Specialized Agents

ccprompts includes 10 specialized AI agents with deep domain expertise. Here's how to use them:

### Example: Rust Development

Get expert Rust assistance:

```bash
# Call the rust-expert agent
/agent rust-expert

# Example task:
"Help me implement a concurrent web scraper with proper lifetime management"
```

**Available agents:**
- `rust-expert` - Ownership, lifetimes, systems programming
- `python-pro` - Modern Python, async patterns, decorators
- `golang-pro` - Goroutines, channels, concurrent systems
- `javascript-expert` - ES6+, async/await, DOM optimization
- `bash-shell-scripting` - Shell automation and scripting
- `fastapi-optimizer` - FastAPI performance tuning
- `documentation-writer` - Technical documentation
- `performance-optimizer` - System-wide optimization
- `systems-architect` - Architecture design patterns
- `agent-template-wizard` - Create custom agents

### Example: Performance Optimization

```bash
# Call the performance-optimizer agent
/agent performance-optimizer

# Example task:
"Analyze and optimize database query performance in my API endpoints"
```

## Safety Features in Action

ccprompts includes enterprise-grade safety for potentially dangerous operations:

### Safe Command Execution

Dangerous commands are automatically detected and executed safely:

```bash
# This command will be executed in an isolated container
./scripts/safe-run.sh "rm -rf temp_files"
```

**How it works:**

<Steps>
  <Step title="Pattern Detection">
    Regex patterns identify potentially dangerous operations
  </Step>
  
  <Step title="Container Isolation">
    Command executes in isolated Ubuntu container with resource limits
  </Step>
  
  <Step title="Read-Only Mounting">
    Project files are mounted read-only for safety
  </Step>
  
  <Step title="Automatic Cleanup">
    Container resources are automatically released after execution
  </Step>
</Steps>

<Warning>
To use the safety system, install Dagger (see [Installation](/installation) guide). The safety system provides 65.7% validation rate across 517+ test cases.
</Warning>

## Tips and Best Practices

<AccordionGroup>
  <Accordion title="Start with Project Analysis">
    Always run `/analyze-project` first to get personalized recommendations based on your specific codebase. The analyzer detects your technology stack and suggests the most relevant commands.
  </Accordion>
  
  <Accordion title="Use Specialized Agents">
    When working with a specific language or framework, invoke the specialized agent for that domain. Agents provide deeper expertise than general commands.
  </Accordion>
  
  <Accordion title="Chain Related Commands">
    Use `/intelligent-chain` for complex workflows. It automatically determines the optimal sequence of commands for your goal.
  </Accordion>
  
  <Accordion title="Install Dagger for Safety">
    Install Dagger to enable the safety system for containerized execution of dangerous commands. This provides enterprise-grade safety with automatic rollback.
  </Accordion>
  
  <Accordion title="Validate Before Committing">
    For repository installations, always run `bun run validate` before committing changes to ensure command structure integrity.
  </Accordion>
  
  <Accordion title="Explore MCP Servers">
    Use `/mcp-discover` to extend Claude's capabilities. With 1000+ available servers, you can add database connections, API integrations, and specialized tools.
  </Accordion>
</AccordionGroup>

## Customization

### Creating Custom Commands

For repository installations, you can create custom commands:

<Steps>
  <Step title="Copy an Existing Command">
    Copy a command from `.claude/commands/` to your desired phase folder
  </Step>
  
  <Step title="Update Structure">
    Update the command structure and content for your use case
  </Step>
  
  <Step title="Add Safety Validation">
    Include safety validation steps in your command
  </Step>
  
  <Step title="Validate">
    Run validation before committing:
    ```bash
    bun run validate
    ```
  </Step>
</Steps>

### Creating Custom Agents

Use the agent template wizard to create specialized agents:

```bash
/agent agent-template-wizard
```

Provide your domain expertise requirements and the wizard will generate a properly structured agent with:
- Consistent 7-step methodology
- Color-coded categorization
- Example use cases
- Validation approaches
